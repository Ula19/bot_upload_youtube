# Цели масштабирования

## Текущее состояние
- SOCKS5 резидентный прокси (primary, ~9 MB/s измерено)
- WARP fallback
- Семафор: 30 параллельных скачиваний
- ThreadPoolExecutor: ~8 потоков
- `_RATE_LIMIT_DELAY = 3s` глобально между скачиваниями
- Rate limit в памяти (бесконечный рост)
- Local Bot API для заливки (14-17 MB/s)

## Параметры резидентного прокси

| Параметр | Значение |
|----------|----------|
| Тип | Static Residential (IPv4) ISP - Dedicated |
| Протокол | SOCKS5 |
| Public IP | 31.133.93.112 |
| Локация | Germany |
| ISP | Liberty Global |
| Speed | 100 Mbit (~12.5 MB/s потолок) |
| Concurrency | 100 threads |
| Bandwidth | Unlimited |
| Uptime | 100% |
| Цена | $2.12/мес |

**Реальные лимиты с этими параметрами:**
- 20+ параллельных запросов — да (упор в bandwidth, не в concurrency)
- 1000-3000 запросов/день — легко
- 5000+/день — один IP может привлечь внимание YouTube → нужна ротация

**Деление bandwidth при параллели:**
- 5 параллельных HD = ~2.5 MB/s каждое
- 10 параллельных HD = ~1.25 MB/s каждое
- 20 параллельных HD = ~0.6 MB/s каждое

## Известные узкие места

| Компонент | Лимит | Решение |
|-----------|-------|---------|
| Прокси bandwidth | 100 Mbit (~12.5 MB/s) | Апгрейд скорости или несколько прокси |
| Прокси concurrency | 100 threads | OK с большим запасом |
| ThreadPoolExecutor | ~8 потоков | Увеличить через `loop.set_default_executor()` |
| FFmpeg merge | CPU-bound | Больше CPU или менее агрессивное склеивание |
| RAM | 8 ГБ | Достаточно до ~30 параллельных скачиваний |
| `/tmp` место | зависит от диска | Фоновая очистка |
| YouTube IP репутация | ~? запросов/день | Ротация IP |

## Метрики для мониторинга

Грепать в логах:
```bash
docker compose logs bot | grep -E "PRIMARY|METRIC|Fallback"
```

Ключевые показатели:
- `[METRIC] get_info` — должно быть < 5s
- `[METRIC] download_video` `speed=` — должно быть > 2 MB/s
- `[METRIC] upload_video` `speed=` — должно быть > 10 MB/s
- Fallback срабатывает редко (< 5% запросов)
